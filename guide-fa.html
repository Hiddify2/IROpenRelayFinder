راهنمای جامع White Proxy (نسخه آفلاین فارسی)

Linux-first Stdlib-only (بدون پکیج پایتون) Offline قابل مشاهده

این سند یک راهنمای کامل و فنی برای استفاده از پروژه White Proxy است: از اجرای اولیه، ساخت IP Pool، اجرای مودهای مختلف، اتصال V2Ray/Xray روی 127.0.0.1:7080، تا توضیح ریزپایپلاین‌ها و رفتارهای پنهان/پس‌زمینه.

این سند بر اساس کد فعلی پروژه نوشته شده است و تمام منوها، گزینه‌ها، فایل‌های خروجی، و منطق داخلی مهم را پوشش می‌دهد.

2) راه‌اندازی سریع

پیش‌نیازها

  • پایتون 3.9 یا جدیدتر
  • برای برخی مسیرها دسترسی sudo/root (به‌خصوص DPI خام و برخی Preflight)
  • اختیاری: masscan و nmap برای اسکن سریع‌تر

اجرای برنامه

python main.py

ترتیب پیشنهادی استفاده روزمره

  1. از منو [1] اسکن بگیر و IP Pool بساز
  2. در صورت نیاز [7] استراتژی DPI را تنظیم کن
  3. با [w] یا [d] یا [m] پروکسی را بالا بیاور
  4. کلاینت‌ها (مرورگر/سیستم/V2Ray) را به 127.0.0.1:7080 وصل کن
  5. برای مشکلات دامنه‌ای از [6] (reroute + ban domain-specific) استفاده کن
در Linux اگر مود DPI را اجرا می‌کنید و موتور raw socket نیاز باشد، برنامه تلاش می‌کند با sudo اجرا شود.

2.1 تنظیم V2Ray/Xray برای Desync با آدرس 127.0.0.1 و پورت 7080

با توجه به اینکه شما کانفیگ‌های Cloudflare Worker را در V2Ray/Xray دارید، کافی‌ست خروجی/ارتباطی که باید از این پروکسی عبور کند را به لوکال پراکسی وصل کنید: Address = 127.0.0.1 و Port = 7080.

دو الگوی رایج اتصال

الگو توضیح چه زمانی مناسب است
HTTP/SOCKS Proxy Chain V2Ray/Xray یا سیستم‌عامل را طوری تنظیم می‌کنید که درخواست‌ها به 127.0.0.1:7080 بروند. وقتی می‌خواهید همه/بخشی از ترافیک از White Proxy عبور کند.
Transparent TLS Path White Proxy بایت اول TLS را تشخیص می‌دهد (مسیر Transparent TLS برای V2Ray/Xray). برای سناریوهایی که ترافیک TLS به شکل شفاف به پروکسی می‌رسد.

چک‌لیست سریع

  • White Proxy را با [d] یا [m] بالا بیاور.
  • در کلاینت خود upstream را روی 127.0.0.1:7080 بگذار.
  • اگر نیاز به DPI injection داری، از منوی [7] استراتژی را انتخاب کن.
  • در Mixed، ترافیک 443 می‌تواند مسیر DPI بگیرد و بقیه/وب عمومی از White Pool استفاده کند.
پروکسی روی چند پروتکل کار می‌کند: SOCKS5، HTTP/HTTPS، و Transparent TLS.

  4. Classification پاسخ: accept / soft_accept / reject / dead
  5. مرتب‌سازی بر اساس score و latency
  6. ذخیره فایل scan و نگه‌داشتن فقط چند فایل آخر

7.1 پایپلاین White Routing (جزئیات کامل)

  1. کلاینت متصل می‌شود (SOCKS5/HTTP/Transparent TLS).
  2. target host/port استخراج می‌شود.
  3. برای دامنه‌های .ir مسیر native مستقیم اعمال می‌شود.
  4. ابتدا exact/wildcard cache بررسی می‌گردد.
  5. اگر cache hit باشد ولی IP برای آن دامنه ban باشد، race اجباری می‌شود.
  6. race همزمان روی target مستقیم + IPهای pool انجام می‌شود (با semaphore).
  7. برنده ثبت/کش می‌شود (برای دامنه‌های غیر-video shard).
  8. در شکست کامل، دامنه به failed log می‌رود و برای TLS می‌تواند 502 برگردد تا خطای cert ظاهر نشود.

نکات عمیق

  • برای دامنه‌های حساس (AI/Google/CDN)، verify_sni سخت‌گیرانه‌تر است (cert + probe).
  • duplicate race برای یک host با race lock جلوگیری می‌شود.
  • fallback race روی banned IPها در صورت شکست primary می‌تواند فعال شود.
  • برای turns.goog fallback alias با SNI گوگل وجود دارد.

7.2 پایپلاین DPI Desync (جزئیات کامل)

  1. مود desync یا mixed فعال می‌شود.
  2. هدف SNI/IP از desync_pairs یا ورودی دستی انتخاب می‌شود.
  3. موتور DPI بر اساس سیستم‌عامل بالا می‌آید (Linux raw socket / Windows WinDivert).
  4. در handshake، seq/ack اتصال مانیتور می‌شود.
  5. ClientHello جعلی با SNI spoof تزریق می‌گردد (یا classic فقط split).
  6. سپس مسیر واقعی relay ادامه پیدا می‌کند.
  7. در شکست‌های سریع پیاپی، auto-rotation استراتژی رخ می‌دهد.
لاگ‌های DPI بعد از 10 ثانیه اول startup به‌صورت خودکار کم می‌شوند (مگر اینکه گزینه همیشه‌نمایش فعال باشد) تا CPU تحت فشار نرود.

7.3 پایپلاین Mixed

در Mixed، منطق White Routing و Desync همزمان فعال هستند: مسیرهای وب/دامنه‌های عمومی می‌توانند از race+pool سود ببرند و مسیرهای TLS هدف‌دار از desync engine استفاده کنند.

  • prewarm/passive/health workers فعال می‌مانند (مثل white_ip)
  • DPI injector هم فعال می‌شود (مثل dpi_desync)
  • برای سناریوی Worker + AI ban domain-specific، Mixed معمولاً بهترین تعادل را می‌دهد

8) ابزارهای پیشرفته (کلیدهای s / c / a)

این ابزارها کیفیت مسیرها و دیتابیس‌های تصمیم‌گیری را در طول زمان بهبود می‌دهند.

8.1 Desync Scanner (کلید s)

خروجی اصلی

  • desync_pairs.json (مسیر write جدید: data/ با migration-aware IO)

پایپلاین

  1. انتخاب لیست SNI (پیش‌فرض / clean_snis / سفارشی).
  2. انتخاب منبع IP (DNS mining و/یا custom).
  3. Resolution چنددوره‌ای DNS و ساخت candidate pairs.
  4. verify_sni همزمان برای هر pair.
  5. ذخیره pairهای معتبر و مرتب‌سازی نتایج.

8.2 SNI Scanner (کلید c)

هدف

پیدا کردن SNIهای clean روی IP تمیز Cloudflare و تولید clean_snis.txt.

منابع IP

  • IP دستی
  • Auto-mine از CNAME
  • Deep scan روی ASNهای Cloudflare ایران
  • Deep scan جهانی Cloudflare

ویژگی مهم

تشخیص DPI redirection با امضاهای محتوایی (مثل peyvandha/cra/10.10.3) انجام می‌شود تا false-clean کاهش یابد.

8.3 Auto-Tuner (کلید a)

Auto-Tuner نرخ‌های Asyncio/Masscan/Nmap را با روش تطبیقی تنظیم می‌کند و در config ذخیره می‌نماید.

منطق کلی

  1. ایجاد baseline کم‌سرعت برای مرجع صحت
  2. کالیبره دقت فعلی موتور
  3. Ramp-up نرخ تا رسیدن به آستانه افت
  4. Binary-like search برای نقطه بهینه
  5. Verify مجدد و اعمال ضریب اطمینان
  6. ذخیره نرخ‌های tuned در config

9) قابلیت‌های پنهان و Passive (خیلی مهم)

1) Passive Background Scanner

  • در white/mixed فعال می‌شود.
  • از ISPهای مرتبط با cache فعلی الهام می‌گیرد و subnetهای جدید را می‌گردد.
  • IPهای جدید را به pool و cache اضافه می‌کند.
  • اگر IP فقط برای Worker مناسب باشد، برای دامنه‌های مشکل‌دار auto-ban می‌کند.

2) Health Checker

  • IPهای pool را دوره‌ای verify می‌کند.
  • IP مرده را به dead pool می‌برد و قابل احیا می‌کند.
  • route cache خراب را prune می‌کند و فایل route را rewrite می‌نماید.

3) Prewarm Routes

در startup برای دامنه‌های latency-sensitive race اولیه می‌زند تا کاربر اولیه delay کمتری ببیند.

4) Auto Pause پس‌زمینه هنگام لود

وقتی تعداد اتصال‌های active از آستانه (پیش‌فرض 6) بیشتر شود، passive/health موقتاً کند یا pause می‌شوند.

5) Purge استارت‌آپ برای media shards

routeهای قدیمی youtube/googlevideo و موارد مشابه در startup پاک می‌شوند تا stale-route تجربه کاربر را خراب نکند.

6) Auto DPI Strategy Rotation

اگر ارتباط‌های DPI خیلی سریع fail شوند، استراتژی خودکار عوض می‌شود (با fallback به classic در failureهای زیاد).

7) Log Throttling

لاگ DPI پس از 10 ثانیه اول startup مخفی‌تر می‌شود تا bottleneck لاگ روی CPU رخ ندهد.

11) معماری فنی (خلاصه Whitepaper)

لایه‌بندی

  1. Entry/Orchestration: main.py, cores/ui.py
  2. Service Layer: app_service, route_service, scan_service
  3. Core Engines: white_core, route_manager, scanner, desync_core
  4. State/IO: runtime_state, config, paths, storage, data_store

الگوی تصمیم‌گیری Route

Direct Rules (.ir / localhost / IP literal)
  -> Cache Hit (exact/wildcard)
    -> Domain-specific Ban Check
      -> Concurrent Race (target + pool)
        -> Persist Winner / Fail Log

Response Classification

  • reject: امضاهای block/captive/region
  • soft_accept: موردهای Worker-like (قابل استفاده محدود)
  • accept: وضعیت HTTP معتبر + footprint محتوایی/سرور
  • dead: timeout/عدم پاسخ یا ساختار نامعتبر

12) عیب‌یابی دقیق

پروکسی بالا می‌آید ولی اتصال برقرار نیست

  • بررسی کن برنامه روی پورت درست گوش می‌دهد (پیش‌فرض 7080).
  • کلاینت واقعاً روی 127.0.0.1:7080 ست شده باشد.
  • در مود d/m اگر injector بالا نیامد، sudo/root یا پیش‌نیاز سیستم را بررسی کن.

دامنه خاص fail می‌شود ولی بقیه خوب‌اند

  • از گزینه [6] استفاده کن تا route آن دامنه reset شود.
  • ban domain-specific باعث می‌شود IP برای بقیه دامنه‌ها قابل استفاده بماند.

اسکن خیلی کند است

  • Auto-Tuner اجرا کن (کلید a).
  • در اسکن، preflight روی masscan/nmap را فعال کن.
  • کیفیت شبکه را در Auto-Tuner درست انتخاب کن (Good/Normal/Shitty).

لاگ خیلی زیاد است

  • گزینه Always Show DPI Logs را خاموش نگه دار.
  • برنامه به‌صورت پیش‌فرض پس از 10 ثانیه لاگ DPI را کم می‌کند.

13) نکات امنیتی/عملیاتی

  • به‌طور پیش‌فرض هیچ dependency پایتون بیرونی لازم نیست (استانداردلایبرری).
  • برای مسیرهای raw packet سطح دسترسی سیستم‌عامل اهمیت دارد.
  • فایل‌های route/cache را دوره‌ای بررسی کن تا stale state نداشته باشی.
  • پس از تغییرات بزرگ، smoke harness را اجرا کن:
python main.py --core smoke
این راهنما به‌صورت کامل آفلاین قابل مشاهده است؛ برای استفاده کافی‌ست همین فایل HTML را در مرورگر باز کنید.
